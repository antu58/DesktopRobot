# Body 技术调研

更新时间：2026-02-20

## 1. 硬件方案调研

### 1.1 主控芯片

- 推荐：ESP32-S3
- 单芯片可行：ESP32-S3-WROOM-1-N8R8 / N16R8
- 双芯片可行（视觉/语音分离）：
  - 视觉芯片：N8R8（摄像头、视觉推理、舵机控制）
  - 语音芯片：N8R2/N8R8（麦克风阵列、VAD、语音前处理）

### 1.2 视觉能力

- 人脸识别与行人检测：ESP-WHO
- 手势与姿态：可选 ESP-DL / TFLite 轻量模型
- 复杂行为：本地轻量分类 + 云端补充

### 1.3 语音能力

- 2~4 麦克风阵列用于声源定位
- 本地 VAD 检测语音段
- 语音段上传云端 ASR（后续接入标准化流程）

### 1.4 执行机构

- 双轴云台（Pan-Tilt）
- 视觉驱动跟踪 + 声学驱动转向的混合控制策略

## 2. 通信与协议调研结论

- 设备通信采用 MQTT，满足轻量、低延迟、弱网重连需求。
- 技能调度模式采用“终端上报能力 + 服务端下发调用 + 终端回执结果”。
- 会话请求采用 HTTP `/v1/chat`，便于与网关、多模态处理链路整合。

## 3. 当前工程落地建议

- 固件侧优先保证 MQTT 稳定性与重连策略。
- 以 `skills` 快照作为能力唯一来源，避免本地/服务端双配置分叉。
- 先稳定 `keyboard_text` 链路，再逐步接入多模态输入。
